Everything that you always wanted to know about the Configuration File:

All program configuration directives are set in the config file.

Configuration files contain several sections that each have the following format:

 [section]
 option=value
…

For example:

  [Query]
query = RANGE
range = 295111,415333
organization = Mozilla
base_url = https://api-dev.bugzilla.mozilla.org/latest

[Database]
database = bugzilla
collection = issues

[Metrics]
m_all = off
m_ttr = on
m_mdr = on
m_ddr = on

[Dashboard]
dashpath = ../dashres/

Section and option names are set by the program.  Option values are the responsibility of the user.  However, values for the query option come from a fixed set, as described on the Query Configuration details page.

Each section is discussed in detail in the pages referenced below.

  • Query Section
  • Database Section
  • Metrics Section
  • Dashboard Section

NOTES:

  1. All configuration information is specified in the configuration file.
  2. Option/Value pair lines can be commented out with the '#' character.  These lines are then ignored by the config file parser.
  3. Option/Value pairs are just that, "pairs".  Don't specify an option and leave the value field blank.  This is an undefined condition.  We'll try to handle it gracefully, but it would be nice if you just didn't do that.
  4. If you don't have an external source for issues, you must have a database containing issue information.
